Roteamento Básico
GET, POST, PUT, DELETE

Rotas são o coração de qualquer API. Elas definem o que acontece quando o cliente acessa uma URL com um determinado verbo HTTP. Aqui você aprende todos os métodos, a lógica REST e como escrever rotas limpas.

Nível 01 — Básico Verbos HTTP REST

01O que é roteamento

Roteamento é o mecanismo pelo qual o Express determina qual função executar para uma determinada combinação de URL + método HTTP.

Pense numa rota como um contrato: "quando alguém fizer uma requisição GET para /usuarios, execute essa função". É exatamente isso que você está declarando quando escreve uma rota no Express.

Anatomia de uma rotaJavaScript
//  método   caminho    handler (req, res)
app.get    ('/usuarios', (req, res) => {
  res.json([])
})

// ↑ app     → instância do Express
// ↑ .get    → método HTTP (verbo)
// ↑ '/...'  → caminho (path/endpoint)
// ↑ handler → função executada na requisição
// Como o Express combina rotas O Express percorre as rotas na ordem em que foram declaradas. A primeira rota que corresponder à URL + método da requisição será executada. Se nenhuma corresponder, a requisição segue para o próximo middleware ou retorna 404.

02Os verbos HTTP — teoria

O HTTP define um conjunto de métodos (verbos) que indicam a intenção da requisição. Cada verbo tem uma semântica específica que você deve respeitar para construir uma API previsível e bem estruturada.

GET
Leitura
Busca um ou mais recursos. Nunca altera dados. Pode ser cacheado.
✓ Idempotente · ✓ Seguro
POST
Criação
Cria um novo recurso. Envia dados no body. Responde com 201.
✗ Idempotente · ✗ Seguro
PUT
Substituição total
Substitui completamente um recurso existente. Envia o objeto inteiro.
✓ Idempotente · ✗ Seguro
PATCH
Atualização parcial
Atualiza apenas os campos enviados. Mais eficiente que PUT.
✓ Idempotente · ✗ Seguro
DELETE
Remoção
Remove um recurso. Responde com 204 (sem body) ou 200.
✓ Idempotente · ✗ Seguro
HEAD
Só os headers
Igual ao GET, mas retorna apenas os headers, sem body.
✓ Idempotente · ✓ Seguro
// Idempotência — entenda de uma vez Um método é idempotente quando fazer a mesma requisição múltiplas vezes produz o mesmo resultado. DELETE de um usuário que já foi deletado ainda retorna "não existe" — o estado final é o mesmo. POST, porém, criaria duplicatas a cada chamada.

03Métodos do Express

O Express expõe um método para cada verbo HTTP. Todos seguem a mesma assinatura: app.METHOD(path, handler).

app.get()

app.get() — buscar recursosJavaScript
// Lista todos os produtos
app.get('/produtos', (req, res) => {
  res.status(200).json([{ id: 1, nome: 'Notebook' }])
})

// Busca um produto específico pelo ID
app.get('/produtos/:id', (req, res) => {
  const { id } = req.params
  res.status(200).json({ id, nome: 'Notebook' })
})

app.post()

app.post() — criar recursoJavaScript
app.post('/produtos', (req, res) => {
  const { nome, preco } = req.body

  // Lógica de criação aqui (banco de dados, etc.)
  const novoProduto = { id: 99, nome, preco }

  // 201 Created — recurso criado com sucesso
  res.status(201).json(novoProduto)
})

app.put() vs app.patch()

Essa é uma dúvida clássica. PUT substitui o recurso inteiro — você precisa enviar todos os campos. PATCH atualiza apenas o que foi enviado.

app.put() — substituição totalJavaScript
app.put('/produtos/:id', (req, res) => {
  // PUT: o body deve conter TODOS os campos do produto
  // Campos não enviados são sobrescritos como undefined/null
  const { nome, preco, categoria, estoque } = req.body

  const produtoAtualizado = {
    id: Number(req.params.id),
    nome, preco, categoria, estoque
  }

  res.status(200).json(produtoAtualizado)
})
app.patch() — atualização parcialJavaScript
app.patch('/produtos/:id', (req, res) => {
  // PATCH: só envia o que quer mudar
  // Ex: { "preco": 999.90 } — apenas o preço muda

  const produtoExistente = { id: 1, nome: 'Notebook', preco: 1200, estoque: 5 }

  // Spread: mantém os dados antigos, sobrescreve apenas os novos
  const atualizado = { ...produtoExistente, ...req.body }

  res.status(200).json(atualizado)
})

app.delete()

app.delete() — remover recursoJavaScript
app.delete('/produtos/:id', (req, res) => {
  const { id } = req.params

  // Lógica de remoção aqui...

  // Opção 1: 204 No Content (mais comum)
  res.status(204).end()

  // Opção 2: 200 com mensagem de confirmação
  // res.status(200).json({ mensagem: `Produto ${id} removido` })
})

04Padrão REST — como nomear suas rotas

REST (Representational State Transfer) é um estilo de arquitetura que define convenções para criar APIs HTTP previsíveis. A ideia central: URLs representam recursos, verbos representam ações.

Método URL Ação Status
GET /produtos Lista todos os produtos 200
GET /produtos/:id Busca um produto pelo ID 200 / 404
POST /produtos Cria um novo produto 201
PUT /produtos/:id Substitui um produto inteiro 200
PATCH /produtos/:id Atualiza campos do produto 200
DELETE /produtos/:id Remove o produto 204
// Erro comum de nomenclatura ❌ /getProdutos · /criarUsuario · /deletar-post/5

✅ /produtos · /usuarios · /posts/5

URLs devem ser substantivos no plural, nunca verbos. A ação já está no verbo HTTP.

Recursos aninhados

Quando um recurso pertence a outro, você aninha as URLs para expressar essa relação:

Recursos aninhadosJavaScript
// Posts de um usuário específico
app.get('/usuarios/:userId/posts', ...)
app.post('/usuarios/:userId/posts', ...)
app.get('/usuarios/:userId/posts/:postId', ...)
app.delete('/usuarios/:userId/posts/:postId', ...)

// Comentários de um post
app.get('/posts/:postId/comentarios', ...)
app.post('/posts/:postId/comentarios', ...)

// Regra: não aninhá mais de 2 níveis (fica ilegível)
// ❌ /usuarios/:id/posts/:pid/comentarios/:cid/respostas

05Route Chaining — app.route()

Quando a mesma URL responde a múltiplos métodos, você pode encadear os handlers com app.route() para evitar repetição e deixar o código mais limpo.

Sem route chaining — repetitivoJavaScript
app.get('/produtos/:id', (req, res) => { ... })
app.put('/produtos/:id', (req, res) => { ... })
app.patch('/produtos/:id', (req, res) => { ... })
app.delete('/produtos/:id', (req, res) => { ... })
Com app.route() — limpoJavaScript
app.route('/produtos/:id')
  .get((req, res) => {
    res.json({ id: req.params.id })
  })
  .put((req, res) => {
    res.json({ mensagem: 'Produto substituído' })
  })
  .patch((req, res) => {
    res.json({ mensagem: 'Produto atualizado' })
  })
  .delete((req, res) => {
    res.status(204).end()
  })

06app.all() e rotas especiais

app.all()

Responde a qualquer método HTTP para um determinado caminho. Muito usado para middlewares aplicados a todas as rotas de um path.

app.all()JavaScript 
// Executado para QUALQUER método em /admin/*
app.all('/admin/*', (req, res, next) => {
  const token = req.get('Authorization')
  if (!token) {
    return res.status(401).json({ erro: 'Token necessário' })
  }
  next() // Continua para a rota correspondente
})

app.get('/admin/usuarios', (req, res) => {
  res.json([]) // Só chega aqui se o app.all() chamar next()
})

Rota 404 — catch-all

Declarando uma rota no final de tudo, você captura qualquer requisição que não correspondeu a nenhuma rota anterior.

Rota 404 — deve ser a últimaJavaScript
// ... todas as suas rotas aqui acima ...

// Catch-all: qualquer URL não encontrada
app.use((req, res) => {
  res.status(404).json({
    erro: 'Rota não encontrada',
    path: req.path,
    metodo: req.method
  })
})
// Ordem importa muito O Express processa rotas na ordem de declaração. Se você colocar um app.all('*') ou app.use() no começo, ele intercepta TUDO. Sempre declare suas rotas específicas primeiro, e handlers genéricos por último.

07Código completo — API de Produtos

Todos os métodos em uma API real, com padrão REST correto e tratamento de erros básico.

API de Produtos — todos os métodosJavaScript
const express = require('express')
const app = express()
app.use(express.json())

let produtos = [
  { id: 1, nome: 'Notebook', preco: 3500, estoque: 10 },
  { id: 2, nome: 'Mouse', preco: 150, estoque: 50 },
]
let nextId = 3

// GET /produtos — listar (com filtro por query)
app.get('/produtos', (req, res) => {
  const { nome, precoMax } = req.query
  let resultado = [...produtos]

  if (nome) resultado = resultado.filter(p => p.nome.toLowerCase().includes(nome.toLowerCase()))
  if (precoMax) resultado = resultado.filter(p => p.preco <= Number(precoMax))

  res.json({ total: resultado.length, dados: resultado })
})

// Usando route chaining para /produtos/:id
app.route('/produtos/:id')

  // GET /produtos/:id — buscar um
  .get((req, res) => {
    const produto = produtos.find(p => p.id === Number(req.params.id))
    if (!produto) return res.status(404).json({ erro: 'Produto não encontrado' })
    res.json(produto)
  })

  // PUT /produtos/:id — substituição total
  .put((req, res) => {
    const idx = produtos.findIndex(p => p.id === Number(req.params.id))
    if (idx === -1) return res.status(404).json({ erro: 'Produto não encontrado' })

    produtos[idx] = { id: produtos[idx].id, ...req.body }
    res.json(produtos[idx])
  })

  // PATCH /produtos/:id — atualização parcial
  .patch((req, res) => {
    const idx = produtos.findIndex(p => p.id === Number(req.params.id))
    if (idx === -1) return res.status(404).json({ erro: 'Produto não encontrado' })

    produtos[idx] = { ...produtos[idx], ...req.body }
    res.json(produtos[idx])
  })

  // DELETE /produtos/:id — remover
  .delete((req, res) => {
    const id = Number(req.params.id)
    const existe = produtos.some(p => p.id === id)
    if (!existe) return res.status(404).json({ erro: 'Produto não encontrado' })

    produtos = produtos.filter(p => p.id !== id)
    res.status(204).end()
  })

// POST /produtos — criar novo
app.post('/produtos', (req, res) => {
  const { nome, preco, estoque } = req.body
  if (!nome || preco == null) {
    return res.status(400).json({ erro: 'Nome e preço são obrigatórios' })
  }

  const novo = { id: nextId++, nome, preco, estoque: estoque ?? 0 }
  produtos.push(novo)
  res.status(201).json(novo)
})

// 404 catch-all — deve ser o último
app.use((req, res) => {
  res.status(404).json({ erro: `Rota ${req.method} ${req.path} não existe` })
})

app.listen(3000, () => console.log('Rodando na porta 3000'))
