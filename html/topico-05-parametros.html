Parâmetros
params, query & body

Toda requisição carrega dados. Eles podem vir na URL, na query string ou no corpo da requisição. Entender a diferença e saber extraí-los corretamente é essencial para construir APIs robustas.

Nível 01 — Básico Dados de entrada Input handling

01Os 3 tipos de parâmetros

No Express existem três lugares de onde você pode receber dados em uma requisição. Cada um tem uma finalidade diferente e é acessado de forma diferente.

https://api.loja.com/produtos/42?ordem=preco&limite=10
https://api.loja.com
Base URL
/produtos/42
req.params → { id: "42" }
?ordem=preco&limite=10
req.query → { ordem, limite }
{ nome, preco }
req.body → no corpo HTTP
req.params
Route parameters
Onde: Na URL
Obrigatório? Sim
Uso: Identificar um recurso específico
Ex: /users/42
req.query
Query string
Onde: Após o ?
Obrigatório? Não
Uso: Filtros, paginação, ordenação
Ex: ?page=2&sort=name
req.body
Request body
Onde: No corpo HTTP
Obrigatório? Depende
Uso: Dados de criação/atualização
Ex: POST com JSON

02req.params — parâmetros de rota

req.params é um objeto com os segmentos dinâmicos definidos na rota com o prefixo :. Eles são parte obrigatória da URL — se não forem fornecidos, a rota não vai corresponder.

Parâmetro simples

Parâmetro únicoJavaScript
app.get('/usuarios/:id', (req, res) => {
  console.log(req.params)     // { id: '7' }
  console.log(req.params.id)  // '7'  ← sempre string!

  res.json({ userId: req.params.id })
})
// GET /usuarios/7  ✓
// GET /usuarios    ✗ (rota não corresponde)

Múltiplos parâmetros

Múltiplos paramsJavaScript
app.get('/usuarios/:userId/pedidos/:pedidoId', (req, res) => {
  const { userId, pedidoId } = req.params

  console.log(req.params)
  // { userId: '5', pedidoId: '12' }

  res.json({ userId, pedidoId })
})
// GET /usuarios/5/pedidos/12

Parâmetro opcional com ?

Parâmetro opcionalJavaScript
// O ? torna o parâmetro opcional
app.get('/arquivos/:nome?', (req, res) => {
  const { nome } = req.params

  if (nome) {
    res.json({ arquivo: nome })
  } else {
    res.json({ mensagem: 'Lista de arquivos' })
  }
})
// GET /arquivos/relatorio.pdf  → { arquivo: 'relatorio.pdf' }
// GET /arquivos                → { mensagem: 'Lista de arquivos' }

router.param() — interceptar parâmetros

O método app.param() permite executar uma função sempre que um parâmetro específico aparece em qualquer rota. Útil para carregar dados ou validar antes de qualquer handler.

app.param() — pré-processamentoJavaScript
// Toda vez que ':userId' aparecer em qualquer rota,
// este middleware é executado primeiro
app.param('userId', (req, res, next, value) => {
  // value = o valor do parâmetro (ex: '42')
  const usuario = { id: Number(value), nome: 'Ana' } // simulando banco

  if (!usuario) {
    return res.status(404).json({ erro: 'Usuário não existe' })
  }

  req.usuario = usuario // anexa no req para usar depois
  next()
})

// Agora req.usuario já está disponível em todas as rotas com :userId
app.get('/usuarios/:userId', (req, res) => {
  res.json(req.usuario) // já carregado pelo app.param
})

app.get('/usuarios/:userId/posts', (req, res) => {
  res.json({ usuario: req.usuario, posts: [] })
})

03req.query — query string

req.query contém os parâmetros da query string — tudo que vem após o ? na URL. São opcionais e usados para filtros, paginação e ordenação, nunca para identificar recursos.

Leitura básica

req.query básicoJavaScript
// GET /produtos?categoria=eletronicos&ativo=true&pagina=2

app.get('/produtos', (req, res) => {
  console.log(req.query)
  // { categoria: 'eletronicos', ativo: 'true', pagina: '2' }
  //                                     ↑ string!   ↑ string!

  res.json(req.query)
})

Query com arrays

Arrays na query stringJavaScript
// GET /produtos?tags=oferta&tags=novo&tags=destaque

app.get('/produtos', (req, res) => {
  console.log(req.query.tags)
  // ['oferta', 'novo', 'destaque']

  // Garantir que sempre seja array
  const tags = [req.query.tags].flat().filter(Boolean)
  res.json({ tags })
})

// Forma alternativa: ?tags[]=oferta&tags[]=novo (mesma coisa)

Paginação — padrão real

Paginação com query stringJavaScript
// GET /posts?pagina=3&limite=20&ordem=recente

app.get('/posts', (req, res) => {
  // Extrair e converter com valores padrão
  const pagina  = Math.max(1, Number(req.query.pagina) || 1)
  const limite  = Math.min(100, Number(req.query.limite) || 10)
  const ordem   = ['recente', 'antigo', 'popular'].includes(req.query.ordem)
                  ? req.query.ordem
                  : 'recente'

  const skip = (pagina - 1) * limite

  res.json({
    pagina, limite, ordem,
    skip,            // quantos registros pular
    dados: []        // resultado do banco aqui
  })
})
// Boas práticas para query string Use query params para: filtros (categoria=livros), paginação (pagina=2&limite=10), ordenação (ordem=preco&direcao=asc), busca (q=notebook), campos específicos (fields=id,nome,preco).

04req.body — corpo da requisição

req.body contém os dados enviados no corpo da requisição HTTP. É o principal veículo de dados em POST, PUT e PATCH. Diferente de params e query, o body não aparece na URL.

// Middleware obrigatório Sem o middleware de parsing, req.body é undefined. Sempre adicione app.use(express.json()) antes de definir qualquer rota que use o body.

JSON body (o mais comum)

req.body com JSONJavaScript
app.use(express.json()) // habilita leitura de JSON

app.post('/usuarios', (req, res) => {
  // Cliente enviou: { "nome": "Carla", "email": "carla@email.com", "idade": 25 }

  console.log(req.body)
  // { nome: 'Carla', email: 'carla@email.com', idade: 25 }
  //                                             ↑ number! body preserva tipos

  const { nome, email, idade } = req.body
  res.status(201).json({ id: 1, nome, email, idade })
})

Form data (urlencoded)

Form HTML / urlencodedJavaScript
// Para dados de formulários HTML tradicionais
app.use(express.urlencoded({ extended: true }))

app.post('/login', (req, res) => {
  // Form enviou: email=joao@email.com&senha=12345
  const { email, senha } = req.body
  // { email: 'joao@email.com', senha: '12345' }

  res.json({ email })
})

// extended: true  → usa biblioteca qs (suporta objetos aninhados)
// extended: false → usa querystring nativo (mais simples)

Usando ambos juntos

JSON + urlencoded juntosJavaScript
// Na prática, você quase sempre usa os dois
app.use(express.json())
app.use(express.urlencoded({ extended: true }))

// Express detecta o Content-Type e usa o parser correto automaticamente:
// Content-Type: application/json          → express.json()
// Content-Type: application/x-www-form-urlencoded → express.urlencoded()
// Diferença: body preserva tipos, query não No req.body (JSON), os tipos são preservados: { idade: 25 } chega como number.
No req.query e req.params, tudo chega como string: "25". Sempre converta explicitamente.

05Conversão de tipos

Como params e query chegam sempre como string, você precisa converter antes de usar em comparações, cálculos ou queries no banco.

Técnicas de conversãoJavaScript
app.get('/produtos/:id', (req, res) => {
  // ── Números ──────────────────────────────
  const id1 = Number(req.params.id)       // '42' → 42
  const id2 = parseInt(req.params.id, 10)  // '42' → 42 (base 10)
  const id3 = +req.params.id              // '42' → 42 (atalho)

  // Cuidado: Number('abc') → NaN  — sempre valide!
  if (isNaN(id1)) {
    return res.status(400).json({ erro: 'ID deve ser um número' })
  }

  // ── Booleanos ────────────────────────────
  const ativo = req.query.ativo === 'true'  // 'true' → true

  // ── Datas ────────────────────────────────
  const dataStr = req.query.data  // '2024-01-15'
  const data = new Date(dataStr)
  if (isNaN(data.getTime())) {
    return res.status(400).json({ erro: 'Data inválida' })
  }

  res.json({ id1, ativo, data: data.toISOString() })
})

06Valores padrão e desestruturação

Uma boa API deve funcionar mesmo quando parâmetros opcionais não são enviados. Sempre defina valores padrão para query params e campos não-obrigatórios do body.

Padrões com desestruturaçãoJavaScript
app.get('/posts', (req, res) => {
  // Valores padrão direto na desestruturação
  const {
    pagina   = '1',
    limite   = '10',
    ordem    = 'recente',
    q        = '',
  } = req.query

  res.json({
    pagina: Number(pagina),
    limite: Number(limite),
    ordem,
    busca: q
  })
})

// GET /posts                   → pagina:1, limite:10, ordem:recente
// GET /posts?pagina=3&limite=5 → pagina:3, limite:5,  ordem:recente
// GET /posts?q=javascript      → pagina:1, limite:10, busca:javascript
Valores padrão no body (PATCH)JavaScript
app.patch('/usuarios/:id', (req, res) => {
  const dadosAtuais = { id: 1, nome: 'Ana', email: 'ana@email.com', ativo: true }

  // Operador nullish coalescing — aceita o novo valor, mas mantém o antigo
  const atualizado = {
    ...dadosAtuais,
    nome:   req.body.nome   ?? dadosAtuais.nome,
    email:  req.body.email  ?? dadosAtuais.email,
    ativo:  req.body.ativo  ?? dadosAtuais.ativo,
  }

  // Ou mais simples: spread sobrescreve apenas o que foi enviado
  const atualizado2 = { ...dadosAtuais, ...req.body }

  res.json(atualizado2)
})

07Padrões úteis no dia a dia

Whitelist de campos do body

Nunca confie que o cliente vai enviar apenas os campos corretos. Extraia explicitamente só o que você precisa.

Whitelist de camposJavaScript
app.post('/usuarios', (req, res) => {
  // ❌ NUNCA faça isso — salva qualquer campo, inclusive "admin: true"
  // db.create(req.body)

  // ✅ Extraia explicitamente os campos permitidos
  const { nome, email, senha } = req.body

  const novoUsuario = { nome, email, senha } // campos controlados

  res.status(201).json(novoUsuario)
})

Combinando os 3 tipos em uma rota

Usando params + query + body juntosJavaScript
// POST /lojas/5/produtos?notificar=true
// Body: { "nome": "Notebook", "preco": 3500 }

app.post('/lojas/:lojaId/produtos', (req, res) => {
  // req.params → identificador do recurso pai
  const lojaId = Number(req.params.lojaId)

  // req.query → opções/configurações da operação
  const notificar = req.query.notificar === 'true'

  // req.body → dados do novo recurso
  const { nome, preco } = req.body

  const produto = { id: 1, lojaId, nome, preco }

  if (notificar) {
    console.log(`Notificando clientes da loja ${lojaId}...`)
  }

  res.status(201).json(produto)
})
// Quando usar cada um — regra de ouro params → "Qual recurso?" — /usuarios/42
query → "Como listar/filtrar?" — /produtos?categoria=games
body → "O que criar/atualizar?" — POST com { nome, preco }

08Código completo — API de Artigos

Uma API real usando os três tipos de parâmetros em conjunto, com validação, conversão e valores padrão.

API de Artigos — params + query + bodyJavaScript
const express = require('express')
const app = express()
app.use(express.json())
app.use(express.urlencoded({ extended: true }))

let artigos = [
  { id: 1, titulo: 'Node.js para iniciantes', categoria: 'backend', publicado: true },
  { id: 2, titulo: 'CSS Grid na prática',         categoria: 'frontend', publicado: true },
  { id: 3, titulo: 'Docker do zero',               categoria: 'devops',   publicado: false },
]

// ─────────────────────────────────────────────────────────
// GET /artigos — com filtros via query string
// ?categoria=backend&publicado=true&q=node&pagina=1&limite=5
// ─────────────────────────────────────────────────────────
app.get('/artigos', (req, res) => {
  const {
    categoria,
    publicado,
    q       = '',
    pagina  = '1',
    limite  = '10',
    ordem   = 'id'
  } = req.query

  let resultado = [...artigos]

  // Aplicar filtros
  if (categoria) resultado = resultado.filter(a => a.categoria === categoria)
  if (publicado !== undefined) {
    const pub = publicado === 'true'
    resultado = resultado.filter(a => a.publicado === pub)
  }
  if (q) resultado = resultado.filter(a => a.titulo.toLowerCase().includes(q.toLowerCase()))

  // Paginação
  const pg = Math.max(1, Number(pagina))
  const lm = Math.min(50, Number(limite))
  const total = resultado.length
  resultado = resultado.slice((pg - 1) * lm, pg * lm)

  res.json({
    total, pagina: pg, limite: lm,
    totalPaginas: Math.ceil(total / lm),
    dados: resultado
  })
})

// ─────────────────────────────────────────────────────────
// GET /artigos/:id — busca por ID (req.params)
// ─────────────────────────────────────────────────────────
app.get('/artigos/:id', (req, res) => {
  const id = Number(req.params.id)

  if (isNaN(id)) return res.status(400).json({ erro: 'ID inválido' })

  const artigo = artigos.find(a => a.id === id)
  if (!artigo) return res.status(404).json({ erro: 'Artigo não encontrado' })

  res.json(artigo)
})

// ─────────────────────────────────────────────────────────
// POST /artigos — criação (req.body)
// ─────────────────────────────────────────────────────────
app.post('/artigos', (req, res) => {
  // Whitelist: extrai apenas os campos permitidos
  const { titulo, categoria, conteudo, publicado = false } = req.body

  // Validação manual básica
  if (!titulo || !categoria) {
    return res.status(400).json({ erro: 'titulo e categoria são obrigatórios' })
  }

  const novo = {
    id: artigos.length + 1,
    titulo, categoria, conteudo,
    publicado: Boolean(publicado),
    criadoEm: new Date().toISOString()
  }

  artigos.push(novo)
  res.status(201).json(novo)
})

// ─────────────────────────────────────────────────────────
// PATCH /artigos/:id — atualização parcial (params + body)
// ─────────────────────────────────────────────────────────
app.patch('/artigos/:id', (req, res) => {
  const id = Number(req.params.id)
  const idx = artigos.findIndex(a => a.id === id)

  if (idx === -1) return res.status(404).json({ erro: 'Não encontrado' })

  // Whitelist de campos editáveis
  const { titulo, categoria, conteudo, publicado } = req.body
  const atualizacoes = {}
  if (titulo !== undefined) atualizacoes.titulo = titulo
  if (categoria !== undefined) atualizacoes.categoria = categoria
  if (conteudo !== undefined) atualizacoes.conteudo = conteudo
  if (publicado !== undefined) atualizacoes.publicado = Boolean(publicado)

  artigos[idx] = { ...artigos[idx], ...atualizacoes }
  res.json(artigos[idx])
})

app.listen(3000, () => console.log('API rodando na porta 3000'))

/* Exemplos de uso:
   GET  /artigos?categoria=backend&publicado=true&pagina=1&limite=5
   GET  /artigos/1
   POST /artigos  →  body: { titulo, categoria, conteudo }
   PATCH /artigos/1  →  body: { publicado: true }
*/
